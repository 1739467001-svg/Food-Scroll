---

### 🔬 技术深度介绍（3分钟）

> "**食笺**采用现代云原生架构，核心设计哲学是**'审美优先，云原生优先'**。
> 
> **前端架构**：
> - React 18 函数组件 + Hooks 设计模式
> - 自定义 `useBreakpoint` Hook 实现响应式布局
> - Tailwind CSS 原子化样式，支持中式传统配色系统
> - Vite 构建工具，秒级热更新
> 
> **后端架构**：
> - Express 中间件模式，JWT 无状态认证
> - SQLite 嵌入式数据库，零配置部署
> - Multer 文件上传，UUID 重名防护
> - 环境变量配置，12 要素应用实践
> 
> **部署架构**：
> - Docker 多阶段构建，镜像体积减少 90%
> - 健康检查端点 `/health`，云平台友好
> - 优雅关闭处理，数据零丢失
> - 支持 Zeabur、Railway、Fly.io、阿里云、腾讯云
> 
> **安全设计**：
> - bcrypt 密码哈希（10轮 salt）
> - SQL 注入防护（参数化查询）
> - CORS 跨域限制
> - JWT Token 过期机制"

### 2. 评委答辩常见问题（Q&A）

**Q1: 这个项目和市面上现有的菜单系统有什么区别？**
> A: 三个核心差异：
> 1. **审美差异**：我们采用中式传统美学设计，区别于市面上千篇一律的模板
> 2. **部署差异**：完全开源，支持私有部署，数据自主可控
> 3. **效率差异**：3小时完成开发，证明现代云原生工具链的极致效率

**Q2: 为什么选择 SQLite 而不是 PostgreSQL？**
> A: 这是经过深思熟虑的架构决策：
> - 对于单餐厅场景，SQLite 完全够用且零配置
> - 嵌入式数据库部署更简单，适合小商家
> - 架构可扩展，需要时可无缝迁移到 PostgreSQL

**Q3: 如何保证数据安全？**
> A: 多层安全设计：
> - 传输层：JWT Token + HTTPS
> - 存储层：bcrypt 密码哈希，SQL 注入防护
> - 部署层：非 root 用户运行容器，环境变量隔离敏感信息

**Q4: 3小时开发是真实的吗？**
> A: 完全真实。这得益于：
> - AI 辅助开发（代码生成、架构设计）
> - 成熟的技术栈选型
> - 模块化的项目结构
> - 所有代码和开发记录都在 GitHub 可查

**Q5: 商业模式是什么？**
> A: 开源免费 + 商业支持：
> - 基础系统完全开源，小商家免费使用
> - 提供付费定制开发服务
> - 提供云托管 SaaS 版本（按量付费）
